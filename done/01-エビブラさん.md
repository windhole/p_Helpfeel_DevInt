# フルリモートワークの Helpfeel では、運動不足と Cosense の読み過ぎに注意 😁

Helpfeel 開発部の新しいメンバーを紹介するインタビューシリーズ「New Developers Interview」の第 1 回。

2024 年 3 月に入社した、エビブラさんです。

エビブラさんは、Helpfeel 開発チームの顧客要望実現ユニットに所属するエンジニア。お客さまからいただいた要望を参考に、新機能を開発したり、既存機能の修正や改善を行っています。

直近では、Helpfeel の問い合わせフォーム機能の機能追加や、Helpfeel の固定 FAQ 表示周りの開発などを手がけています。

インタビュアーは、Helpfeel CTO の秋山さん。

// 画像による小見出し
# 学生時代にScrapboxを使っていたので求人を見たとき「この会社、 知ってるぞ!」と思いました。


秋山さん：最初に、Helpfeel に入社するまでの経緯を話してもらえますか？

エビブラさん：学校は、IT 系の専門学校に通っていました。専攻は IoT だったのですが、独学で PHP を勉強して Web 開発を始めて、そこから Web エンジニアを目指すようになりました。

卒業して最初に就職したのは SES の会社だったのですが、私は社内で自社開発のプロジェクトをやっていました。2 社目も SES で、そこでは外注先へ行っていたのですが、社内で作っていた SaaS の開発を担当しました。

秋山さん：そこから Helpfeel に転職されたわけですが、Helpfeel を選んだ決め手は何でしたか？

エビブラさん：学生時代に Scrapbox（現：Cosense）を使っていたので、求人を見たとき「この会社、知ってるぞ！」と思いました。Scrapbox が使いやすかったのもあって、すごい会社だなと思っていました。

秋山さん：おお、ありがとうございます（笑）。で、実際に入社してみて、どんなことを感じましたか？

エビブラさん：Helpfeel は、これまでの会社とはだいぶ違いましたね。toB の開発は、前の会社でもやっていたのですが、そこでは、PdM が仕様を議論して決めて、そこからエンジニアにタスクが降ってくる、みたいな感じでした。

でも Helpfeel では、仕様の議論のところからエンジニア中心で回っていて、ボトムアップみたいな感じで開発が進んでいきます。全然違うんだなと、最初に思いました。

// 画像による小見出し
## フルリモートワーク　実際にやってみてどうですか？　 Cosense があるので大丈夫ですね。

秋山さん：いいですねぇ。エビブラさんから見て、Helpfeel はどんな感じに見えていますか？

エビブラさん：検索が好きな人が多いですね（笑）。僕は、検索は面白いと思うけど、あまり得意じゃないので。先日も出張で一緒になったメンバーが検索アルゴリズムの話で盛り上がってましたが、何を言ってるかまったく分からなかったです（笑）。

開発部以外では、みんな個性を持ってるという感じですね。All Hands Meeting（毎週の全社ミーティング）での発表でも面白いことをしている人が多いですし。毎週毎週、いろんな人の話が個性的で面白いですね。

秋山さん：Helpfeel はフルリモートワークですが、実際にやってみてどうですか？

エビブラさん：以前の勤務先は、週 1 回出社のハイブリッドだったのですが、そのときは、週 1 で出社して対面で話す機会があるのは良いなと思ってました。Slack だけだと、何かふわっとした話になって、あとで対面で聞きたいと思うようなことがあったりしましたから。

でも Helpfeel では Cosense があるので、そういった Slack だけのときの課題がなくて。Cosense を探せばだいたいのことは出てくるので、週 1 ぐらいで対面で話したいというのがないんですよね。なので、フルリモートは、体だけ気をつければ大丈夫。

秋山さん：確かに。健康大事ですね。運動不足にならないようにしないと（笑）。Cosense の話が出ましたが、社内の Cosense プロジェクトは、今、9 万ページぐらいありますね。

エビブラさん：社内の Cosense は、読んでて楽しいですよね。ついつい、出てくるリンク先を辿って読みふけって、あっという間に 1 時間、2 時間と時間が溶けちゃうことがあります。

秋山さん：Helpfeel あるあるですね。仕事のことを調べようと思ったのに、気づいたらラーメンのページを見てるとか（笑）。

エビブラさん：Fake がなくて、信用できる情報だけなので、読んでてタメになることも多いです。だから、ついつい読んじゃう（笑）。

## エンジニアもプロダクト開発に主体的に関わっているので、自分が独立したときにも有用な経験が積めています。

秋山さん：Helpfeel のエンジニアに多いのですが、エビブラさんも個人開発をされてますよね？

エビブラさん：はい。最近だと、飲食店向けのメニュー作成アプリを作ってました。以前、セルフオーダーのシステムを作ったことがあるのですが、それで、海外から来日する観光客にとってメニューに課題があることに気づきました。メニューの言葉も分からないし、写真がないとどんな料理かも分からないので。多言語対応メニューをサクッと作れるようになって、お店の手助けになればと。

秋山さん：いい話ですね。エビブラさんが、やりがいを感じるのはどんなときですか？

エビブラさん：結構自由に振る舞えるところですね。私は Helpfeel（プロダクト）の開発チームですけど、（別プロダクトの）Cosense のコードを読んで質問したり議論したりもできます。そういうところで「壁」がないのは、すごいと思いました。自分が動けるところが広くて、自由度が高いので、その辺にやりがいを感じます。

これまでだと、エンジニアに割り振られたことをやるだけだったので、「これをやったらもっと良いんじゃないか」みたいな発想にならないんですよね。

でも Helpfeel だと、こういう要望があるというのが来たら、それをどうやって実現するかは、エンジニアも一緒になって議論し、UI デザイナーさんにも相談したりしながら作り上げていきます。エンジニアもプロダクト開発に主体的に関わっているので、今後、もし自分が独立したときにも有用な経験が積めてる感じがします。

秋山さん：それは嬉しいですね。エビブラさんは、将来こんな風になりたいというイメージはありますか？

エビブラさん：できれば独立したい気持ちは結構あります。自分のサービスを作って一発当てたい。でも、自分はあまり管理職には就きたくないというか無理だろうなと思っていて、開発とか技術は好きなので、できれば、そういうことにずっと関わっていたいと思ってます。

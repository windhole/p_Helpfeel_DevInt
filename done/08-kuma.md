# 「これまで経験しなかった、まったく新しい課題」を求めてHelpfeelへ


Helpfeel開発部の新しいメンバーを紹介するインタビューシリーズ「New Developers Interview」。第8回は、熊澤 裕治さんです。

熊澤さんは、顧客要望実現チームに所属するプロダクトマネージャー（PdM）。Helpfeel FAQに対するお客さまからのご要望を、いつ、どのような形で製品に取り入れるのかを検討し、実際の企画や開発へとつなげていく役割を担っています。

インタビュアーは、Helpfeel CTOの秋山さん。


// 画像による小見出し
最初に入った銀行では
インターネットバンキングの
企画の仕事を10年ほど担当


秋山さん：最初に、熊澤さんのこれまでのキャリアについて、教えてもらえますか？

熊澤さん：新卒で、銀行に入ったのですが、当時は将来について深く考えてなくて、「広告宣伝って、何だか華やかで面白そう」と思っていました。

秋山さん：広告宣伝に興味があったんですね。

熊澤さん：特別にやりたいことがあったわけではなくて、とにかく派手そうだからと、広告宣伝に惹かれたという感じです。

秋山さん：なるほど。

熊澤さん：広告宣伝に興味を持って入社したものの、配属されたのは、銀行の中ではメインストリームではない、非対面チャネルを担当する部署でした。具体的に言うと、コールセンターやATM、インターネットバンキングなどを担当する部門です。私は主に、インターネットバンキングのビジネスサイド、つまり企画の仕事を約10年ほど担当しました。

銀行というところは、通常、2、3年ごとに異動があるものなのですが、私の場合は一度も異動せず、部署名は変わりつつも、ほとんど同じ仕事を続けてきました。

秋山さん：面白いですね。銀行でも異動しないパターンがあるんですね。

熊澤さん：私の同期が新卒で180人ぐらいいましたが、異動しなかったのは、その中の3人だけでした。

秋山さん：専門性高すぎて、あんまりガチャガチャいじれない、みたいな感じに近い？

熊澤さん：そうですね、私以外の他の2人もそうですけど、あまりにも深く入り込み過ぎて、もう異動させたくてもさせられないという感じで。

秋山さん：そういうこともあるんですね。

スタートアップ業界だと、2、3年で転職される方もいたりしますが、熊澤さんは銀行に10年いたというのは、腰を据えてガッツリお仕事されてた感じなんですか？

熊澤さん：何度か転職しようと思ったんですけど、その都度、炎上プロジェクトに放り込まれるので、なかなか辞めどきがなく……。

その後、転職を考えるタイミングで、ちょうど担当していたプロジェクトが一区切りついたので、本格的に転職活動を始めました。


// 画像による小見出し
転職を考えたのは
自分の会社の未来を
語れなかったから


秋山さん：最初から、PdMだったんですか？

熊澤さん：実は、そうでもなくて。

転職した会社のCTOから「大規模プロジェクトの経験があるなら、プロジェクトマネージャー（PjM）として入ったほうがいい」と提案されまして、開発部や技術部など、いわゆる開発組織に所属することになりました。

そこでPjMとして仕事をしていたのですが、次第にプロダクト全体を見るような役割へとシフトしていき、途中からPdMとしての業務も任されるようになりました。そこからキャリアがPdMに移っていって、2社目以降、主にベンチャー企業やスタートアップのような組織でPdMの経験を積んできました。

秋山さん：銀行出身でベンチャーに移るというのは、大きな決断だったのかなと思うのですが、ベンチャーやスタートアップ業界に行こうと思ったきっかけは何だったんですか？

熊澤さん：銀行に、すごく閉塞感を感じていて……。

最初にいた銀行は、誰も自分の会社の未来を語れなかったんですよね。

秋山さん：おおー、なるほど。

熊澤さん：公式には、10年後は良くなります、と言うんですけど、本音でそう言ってる人が誰もいなくて。これはちょっとまずいな、と。

ちょうど「来年は管理職だね」と言われてたときだったのですが、自分が部下を持ったときに「こういうビジョンに向かってやってくんだよ」と語れないことに、モヤモヤした気持ちがありました。

このままでは自分がダメになってしまいそうだと感じて、当時キラキラしていたベンチャーの世界に飛び込めば、何か新しい変化があるかもしれない、と。

秋山さん：今でこそスタートアップは、会社選択のひとつとして一般的になってきましたが、当時はまだ「キラキラしていて、勢いがあるけど、謎なところ」みたいな印象が強かったですよね。

熊澤さん：そうですね。

私がスタートアップに入った当時は、創業初期というわけではなかったのですが、まだ「キラキラしている」「オフィスが綺麗で、ソファに寝転びながら仕事している」みたいなイメージでした（笑）。


// 画像による小見出し
Helpfeelなら
これまで経験しなかった
まったく新しい課題がありそう


秋山さん：最初は業務委託という形でしたが、Helpfeelを選んでいただいた決め手は何でしたか？

熊澤さん：前の会社を辞めて業務委託で仕事を探し始めたとき、いくつかの会社からお声がけをいただいていました。その中で、私としては「フルリモートワークで働ける」という点が大きな魅力でした。

もうひとつ良かった点は、Helpfeelでは他社とは違って、「見たことがあるような課題」ではなく、これまで自分が経験してこなかった「まったく新しい課題」がありそうだと感じたことです。

他の会社だと「この課題はもう何度も経験してきたな」という内容が多く、お声がけいただく理由もそこにあるのかなと思ったのですが、正直なところ、少し飽きてしまっている部分もありました。なので、違った課題に取り組めそうなところに魅力を感じたのが大きかったです。


秋山さん：当時、特に「これは」と感じたのは、どんなところでした？

熊澤さん：募集要項などに明確に書かれていたわけではないのですが、プロダクトを作るチームは、基礎的にエンジニアがプロダクトマネジメント的な視点も持ちながら開発している印象を受けました。一方で、Helpfeelとして、エンタープライズ向けにどんどん力を入れていこうというメッセージも感じられて、そこに少しギャップがあるのかなと感じました。

私はこれまで、エンジニアリングとビジネスサイドの組織をつなぐ役割を長くやってきたので、まさにその橋渡しの部分で貢献できるのでは、と思いました。

秋山さん：当時のHelpfeelは、良くも悪くもプロダクトアウト、いわゆるシーズ思考が少し強めな会社だったと思います。その中で、エンタープライズ領域のリアルな課題にプロダクトを実際に届けていく、という部分は、まだまだ強化すべき点が多かったんですよね。

熊澤さんが入社してくれてからは、プロダクトマネジメントの日々の業務だけでなく、もっと一段階上の体制づくりの部分まで力強く進めてくれていて、本当に助かっています。今振り返ると、確かにそこが大きな課題だったなと改めて感じます。

熊澤さん：ありがとうございます。


// 画像による小見出し
エンジニアも
課題解決に前向きなので
コミュニケーションで
悩むことがない



秋山さん：熊澤さんは、業務委託から正社員に転換していただきました。

一度、業務委託という形で関わっていただいたことで、Helpfeelの良い面も課題となる部分も、いろいろ見えていたのではないかと思います。その上で、私たちから「フルタイムで働いてみませんか」とお声がけした際に、入社を決めた理由や、決め手となったポイントは何でしたか？

熊澤さん：入社の決め手は、率直に言うと、コレというものがあったわけじゃないんですよね。

秋山さん：おお、そうなんですね。

熊澤さん：一言で言うと、非常にストレスなく働けている、というのが大きいですね。

これまでの職場では、エンジニアの方々とのコミュニケーションで、どうしても認識のズレや、目線を合わせることに苦労することがたびたびありました。もちろん、全員というわけではありませんが、一部そういった方もいて、コミュニケーションの部分で悩むことが多かったです。

でも、Helpfeelではそういった悩みがほとんどありません。エンジニアの皆さんはそれぞれバックボーンが違うのですが、会話の中で基本的に皆さんポジティブですし、「そんなこと言われても困る」といったネガティブな反応をされることがまずありません。実際に課題解決のために「じゃあどうしようか」「こうしたら良いのでは」という前向きな議論が多く、そこがとてもいいなと感じています。

PdMとして日々もっとも多く時間を使うのがエンジニアとのやり取りなので、そのコミュニケーションでストレスがないことは、非常に大きな魅力でした。

秋山さん：なるほど。私が言うと手前味噌になってしまうのですが、Helpfeelのエンジニアは、プロダクトマネジメント領域に対する興味だったり、理解があるメンバーが多いという自負があります。そういうところも関係しているんですかね？

熊澤さん：そう思います。

課題に対して真剣に向き合い、「どうやって乗り越えるか」「よりよいものをどうやって提供するか」といった意識を持っているメンバーが多いと感じます。誰かから言われたから機能を3カ月で作る、といったことを重視するのではなく、リアルな課題に対してどう向き合い、どう解決していくかを一緒に考えられるのが前提としてあるチームですね。

秋山さん：いいですね。

顧客の課題に率直に向き合うみたいなところは、昔からいる人だけでなく、最近入ってきた人も含めて、みんな意識はかなり強いですよね。

熊澤さん：そうですね。

秋山さん：私も一緒に仕事していて、メンバーに対して、気持ち良さというか、爽やかさを感じるなというところはありますね。ありがとうございます。


// 画像による小見出し
「これはちょっと難しいかも」
と思いながら投げたボールでも
しっかり受け止めてくれる
その安心感がありがたい


秋山さん：実際に働いてみて、入社後にイメージが違ったという部分はありましたか？　入社前や業務委託で関わっていたときと比べて。

熊澤さん：エンジニアチームのレベルの高さには、良い意味で、とてもギャップを感じました。

先ほどコミュニケーションの話をしましたが、その点も含めて、本当に優秀な方が多いなと日々感じています。採用資料などにも「エンジニアチームのレベルが高い」と書かれていましたが、実際に入ってみて、そのことを実感します。他社ならトップエンジニアとして活躍していそうな方が何人もいるので、本当に心強いです。

また、チームとしてのやり取りもスムーズで、自分が「これはちょっと難しいかもしれないな」と思いながら投げたボールでも、しっかり受け止めてくれるんです。その安心感や頼もしさが、働く上でとてもありがたいなと感じています。

秋山さん：いいですね。

熊澤さん：会社全体としては、いま、すごく数字が伸びている時期なので、もっとこう、イケイケなのかと思ってました（笑）。でも実際には、オラオラ体育会系な感じでもなかったので、意外だなと感じました。

秋山さん：なるほど。体育会系でないとすると、Helpfeelは何系なんでしょう？

熊澤さん：なんでしょうね……。なんか、ネガティブな意味ではなく、まったりしてるなって思いました、最初、会社に入ったときは。

秋山さん：確かに。

コミュニケーションとして、「激詰めする文化」というよりは、シンプルに「課題に対して一緒に頑張って向き合おうね」みたいな「まったりさ」はありますよね。

熊澤さん：はい。激詰めされてるのは見たことないです（笑）。

秋山さん：一方で、課題に向き合うとか、事業成長を作っていくところでは、真剣に取り組んでるところもありますしね。

熊澤さん：入社直後は、そこまで踏み込んだシビアな議論に参加することはありませんでしたが、最近は、例えばプロダクトの戦略についてなど、より真剣な、シビアな話し合いをする場面も増えてきました。やるべきところはしっかりと引き締めて議論する、というメリハリがあるなと、最近は実感しています。



// 画像による小見出し
自分が話した言葉を
他のメンバーが自分の言葉として
会話の中で使ってくれたとき
苦労が報われたと感じます


秋山さん：熊澤さんにとっての、実際に働く中での「やりがい」や、仕事の醍醐味、面白さについて教えていただけますか？

熊澤さん：PdMという立場上、ここが課題だと考え、「こういう体験を提供すれば、お客様の行動が変わるのでは」「価値を感じていただけるのでは」と思い描いて進めたことが、実際にその通りのフィードバックをいただけたり、実際にご利用いただけたりした経験は、一番のやりがいだと感じています。そういった瞬間には、仕事の面白さを実感しますね。

また、チームに対してで言えば、自分なりに考えて言葉やコンセプトに落とし込んだものが、徐々にメンバーにも広がっていき、気がつくと、自分が発した言葉を他のメンバーが自分の言葉として会話の中で使ってくれているのを見ると、それまで言葉を練り、考え抜いてきた苦労が報われるような気持ちになります。

PdMは、組織図とは異なるレイヤーで、ある種リーダー的な立場として振る舞う必要があると思うのですが、そういった役割の中で、自分の考えや言葉がメンバーに浸透し、影響を与えられたと感じたときには、本当に面白さを感じます。

秋山さん：いいですね。

いま、熊澤さんから2つのポイントについてお話しいただきました。まず1つ目の、「課題を発見して解決まで導く」という醍醐味について、もう少し詳しく教えて頂けますか？

熊澤さん：この会社では、「この課題の、この部分だけを解いてほしい」といったような、明確に限定されたお題を与えられることは、ほとんどありません。むしろ、もっと幅広く任せてもらえていると感じています。

その中で、自分なりにたくさんの案を出してはボツにしたりして、いろいろと試行錯誤を重ねていきます。多くの案は最終的に形にならず終わることも多いですが、そんな中でも「これだ」と思えるものに出会い、そこから解決策まで一貫して取り組める環境があるので、とてもやりがいを感じますし、楽しいですね。

秋山さん：ありがとうございます。いいですね。

熊澤さんの練度の高さと一致した課題解決みたいなところがあって、何かこう、幅広に見られる能力と胆力があるからこその面白さ、という面もあるのかもしれないですね。

熊澤さん：そうですね。人にもよると思うのですが、練度が高くて、かつ本人に「もっとやってみたい」という意欲や希望があれば、本当に幅広い領域を任せてもらえます。やる気がある方にとっては、とても楽しい環境だと思います。


//画像による小見出し
PdMの仕事の中で
言葉や意義をしっかり伝えることは
大きな割合を占めている


秋山さん：2つ目のポイントとして挙げていただいた、コンセプト浸透の話も詳しくお聞きします。

熊澤さんが言葉を練って、それがチームの中に浸透し、他のメンバーが自分の言葉として使ってくれる──その瞬間がとても象徴的で、「リーダー」としての大事な役割なのかなと感じました。リーダーがどのように言葉を伝えていくかで、チームのモチベーションや思考の方向性が大きく変わると思うのですが、「言葉」というものを大切にする姿勢は、これまでもずっと意識されてきたのでしょうか？

熊澤さん：そうですね。

最初の頃は、それこそPjMだったときも、何十人もいるメンバーの認識を揃えたり、課題をちゃんと整理したりするために、言葉の大切さは実感していました。みんながきちんと分かる粒度にまで言葉を練り上げて、それを適切な場で伝えるというところは、苦労もしてましたし、こだわっていたところもあります。そこはPdMになっても変わってないかなと思います。

秋山さん：これからPdMを目指したい人にも、とても参考になるお話ですね。どう伝えるかの技術みたいなところも含めて。

熊澤さん：PdMの仕事の中で、言葉や意義をしっかり伝えるというのは、本当に大きな割合を占めていると思います。

いま私が担当しているのは、新規のMVPを少人数で作るフェーズではなく、すでに形ができあがった既存事業をどう伸ばしていくかというフェーズです。

その分、関係者の数も多く、また影響するシステムや業務も広範囲に渡ります。そのため、なぜこの取り組みをするのか、どんな意義があるのかをきちんと伝えられないと、どこかの段階で細部がずれてしまうことがあります。だからこそ、そういった伝え方や意義づけには、特に意識して丁寧に取り組むようにしています。


秋山さん：うん、なるほど、良い話ですね。ありがとうございます。これまでのご経験や積み重ねが、そういった熊澤さんの考え方を形作ってきたのだと感じました。


// 画像による小見出し
一番影響を受けたのは寺本さん
圧倒的に高い技術力に加えて

秋山さん：一方で、Helpfeelに入社してから、新たに得た知見や発見、印象深い気づきなどはありましたか？　もしあれば、教えてください。

熊澤さん：はい、ありました。実名を出しちゃうんですが、一番影響を受けたのは、寺本さんです。

寺本さんが新規ソリューション開発に取り組む姿を見ていると、圧倒的に高い技術力に加えて、発想力や提案の自由さが、本当に際立っていると感じます。

秋山さん：（大きく頷く）

熊澤さん：寺本さんを見ていると、自分の思考は、まだまだ自由になりきれてなくて、制約がかかっているなと実感させられます。

寺本さんは非常に自由な発想をされる方で、「なるほど、そういう考え方やアプローチもあるのか」と学ばさせていただく場面が多いです。

一方で、しっかりと押さえるべきポイントや常識的なラインも絶対に外さないので、その自由さと、守るべき部分のバランス感覚が本当に絶妙だと感じています。寺本さんの存在は、私にとって非常に刺激的で、多くの学びを与えてくれていると思います。自分ももっと思考の制約を外して、柔軟に物事を考えていかなければいけないと感じます。

秋山さん：なるほど。いや、これ、面白い話ですね。

寺本さんは、Helpfeelの中でも「THEハッカー」と呼べる存在だと思います。ただ、そのバランス感が本当に独特で、決してすべてが平均的というわけではなく、すべての要素がしっかりと尖っている、そんなバランスの良さを感じますよね。

熊澤さん：そうですね、尖ってる……そう、尖ってるところは圧倒的に尖ってると思うんですけど、何か、ただの変人では終わらずにちゃんと最後はまとめられる……というか。

秋山さん：確かに。

熊澤さん：ただ尖ってるだけじゃないところがすごい。なのに、尖ってるところはちゃんと失われてない、というか。そこがすごいなと思いますね。

秋山さん：そのユニークネスとリアルワールドのデプロイという繋ぎ込みが、やっぱり寺本さんは圧倒的ですよね。

熊澤さん：そうですね、なかなか、あのパラメータの絶妙な振り分け方がすごいなと思いますね。
